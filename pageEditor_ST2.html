Home Page

Technical Description

The Home Page of the website is implemented using HTML and CSS to ensure a structured, visually appealing, and user-friendly experience. The page utilizes:
Semantic HTML: Tags such as header, nav, main, section and footer provide clear content structure, improving readability and SEO.
Navigation Bar (nav): Contains an unordered list (ul) with links (a) for seamless navigation across pages.
Content Section: Uses a combination of div containers and CSS flexbox/grid properties to arrange content responsively.
CSS for Layout and Styling:
max-width, padding, and margin ensure optimal content alignment.
flexbox is used for a responsive design that adapts to different screen sizes.
font-family and color properties create a visually appealing theme.
Image Optimization: Images are assigned appropriate alt attributes and width/height properties to enhance performance.

Accessibility

To ensure accessibility, the following measures are implemented:
Alt Text for Images: All images have meaningful alt attributes to assist screen readers.
Keyboard Navigation: The navigation bar and interactive elements can be accessed via the keyboard (tab key support).
High Contrast and Readable Fonts: Adequate color contrast and rem/em font sizes improve readability.
ARIA Roles and Landmarks: Used in key sections (role="navigation", role="main") to enhance screen reader support.

Table

Technical Description

The Table Page is designed to display structured data on sustainable city initiatives. Key implementation details include:

HTML Table (table): Used to structure information into rows (tr) and columns (th, td).

Responsive Design: CSS media queries hide specific columns on smaller screens to improve readability.

overflow-x: auto ensures horizontal scrolling when necessary.

Table Styling: border-collapse keeps table borders clean and structured. nth-child(even) applies alternating row colors for better distinction.

Hover effects (:hover) provide a subtle interaction cue.

Navigation and Branding: The page follows a consistent structure with a header containing the logo and navigation links, ensuring uniformity across the site.

Challenges and Lessons Learned

The navigation bar had inconsistencies across different pages due to CSS conflicts.
Solution: Ensured that the global CSS file applied a consistent style and standardized spacing and alignment.

Some text, especially within table headers, lacked contrast against the background, making it harder to read.
Solution: Adjusted text colors to improve readability while maintaining the pageâ€™s visual aesthetic.

Initially, the table did not display properly on smaller screens, causing text overflow and misalignment.
Solution: Implemented CSS media queries to hide less critical columns on mobile devices and added a scrollable container to ensure usability.

Mobile-First Design is Essential Designing for smaller screens first ensures a better user experience on all devices.
Consistency in CSS Saves Time Maintaining a shared stylesheet prevents unnecessary styling conflicts across different pages.
Accessibility Should Be Considered from the Start Implementing semantic HTML and ensuring color contrast from the beginning prevents major fixes later.

Compliance

Color Contrast: The text must have sufficient contrast against the background, making it readable for users with visual impairments.
Keyboard Navigation: Ensure that all interactive elements, such as forms and navigation menus, are fully navigable using the keyboard for users who cannot use a mouse.
Alt Text for Images: All images, including those on the homepage and content pages, should have descriptive alt text to assist screen reader users.
Semantic HTML: The use of proper heading structures (e.g., h1, h2, h3) helps users navigate the page content and is essential for screen readers.
